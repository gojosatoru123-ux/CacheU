# Design Patterns

## Creational
### Singleton
- Single instance
- Global access point
- Lazy initialization
- Thread safety needed
### Factory Method
- Subclass decides type
- Decouples creation
- Open/Closed friendly
### Builder
- Fluent chain API
- Complex objects
- Director optional
### Abstract Factory
- Family of products
- Platform independence

## Structural
### Adapter
- Bridge incompatible
- Wrap legacy code
- Interface translation
### Decorator
- Add behavior
- Runtime wrapping
- Avoids subclassing
### Facade
- Simplify interface
- Hide complexity
- Single entry point
### Proxy
- Control access
- Lazy loading
- Caching layer

## Behavioral
### Observer
- One-to-many events
- Pub/Sub pattern
- Loose coupling
### Strategy
- Swap algorithms
- Runtime selection
- Eliminates if/else
### Command
- Encapsulate actions
- Undo / Redo
- Queue operations
### Iterator
- Traverse collections
- Uniform interface
